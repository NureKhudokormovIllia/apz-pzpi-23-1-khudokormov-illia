3 ОПИС ВИКОНАНОЇ РОБОТИ
3.1 Загальні відомості про шаблони (патерни) проєктування
Шаблон (патерн) проєктування - це усталене, перевірене практикою рішення типової задачі, яка виникає при проєктуванні програмного забезпечення. Патерни не є готовими класами або бібліотеками, які можна вставити у програму, а описують загальні підходи до вирішення повторюваних проблем.
Фундаментальна систематизація патернів проєктування об’єктно-орієнтованого програмування була здійснена у 1994 році у книзі «Design Patterns: Elements of Reusable Object-Oriented Software» групою авторів: Еріхом Ґаммою, Річардом Гелмом, Ральфом Джонсоном та Джоном Вліссідесом (відомих як «Bank of Four», GoF).
Патерни GoF класифікуються за метою на три групи: породжувальні (Creational) - 5 патернів, що відповідають за механізми створення об’єктів; структурні (Structural) - 7 патернів, що описують способи утворення структур із класів та об’єктів; поведінкові (Behavioral) - 11 патернів, що відповідають за взаємодію та розподіл обов’язків між об’єктами.
3.2 Опис обраного патерна
Назва патерна: Спостерігач (Observer).
Тип шаблона: поведінковий (Behavioral).
Альтернативні назви: Publish-Subscribe, Dependents.

Основне призначення патерна Observer полягає у визначенні залежності «один до багатьох» між об’єктами: при зміні стану одного об’єкта (суб’єкта) усі залежні від нього об’єкти (спостерігачі) автоматично сповіщаються та оновлюються.
Патерн Observer доцільно застосовувати у таких ситуаціях: зміна стану одного об’єкта потребує відповідних змін в інших, при цьому невідомо, скільки об’єктів потребує змін; об’єкт має повідомляти інші об’єкти, не знаючи, хто вони; реалізація розподіленої системи обробки подій; побудова шаблону MVC, де модель повідомляє представлення про свої зміни.
3.3 Структура шаблона (патерна)
Патерн Observer включає такі структурні компоненти:
* Subject (Суб’єкт) - інтерфейс або абстрактний клас, що визначає методи attach(), detach() та notify() для управління спостерігачами та їх сповіщення.
* Observer (Спостерігач) - інтерфейс або абстрактний клас, що визначає метод update() для отримання сповіщень від суб’єкта.
* ConcreteSubject - конкретна реалізація суб’єкта, яка зберігає стан та список зареєстрованих спостерігачів. При зміні стану викликає notify().
* ConcreteObserver - конкретна реалізація спостерігача, що реалізує метод update() для синхронізації власного стану зі станом суб’єкта.

Взаємодія об’єктів відбувається за такою схемою: клієнтський код реєструє спостерігачів у суб’єкта через метод attach(). При зміні свого стану суб’єкт викликає notify(), яка послідовно викликає update() для кожного зареєстрованого спостерігача. Спостерігачі можуть отримувати актуальний стан суб’єкта через метод getState().


3.4 Реалізація шаблона (патерна)
Реалізацію патерна Observer виконано мовою програмування Python з використанням модуля abc (Abstract Base Classes) для визначення абстрактних інтерфейсів.
Основні етапи реалізації:
1) Визначення абстрактних інтерфейсів Observer та Subject з використанням ABC.
2) Реалізація класу ConcreteSubject зі збереженням стану та списку спостерігачів.
3) Реалізація класу ConcreteObserver, що реагує на зміни стану суб’єкта.
4) Демонстрація використання: реєстрація, зміна стану, автоматичне сповіщення.

В.2 Програмний код патерна Observer (observer.py)
 1  from abc import ABC, abstractmethod
 2  from typing import List
 3  
 4  # Інтерфейс спостерігача
 5  class Observer(ABC):
 6      @abstractmethod
 7      def update(self, subject: 'Subject') -> None:
 8          """Метод оновлення стану спостерігача."""
 9          pass
10  
11  # Інтерфейс суб'єкта
12  class Subject(ABC):
13      @abstractmethod
14      def attach(self, observer: Observer) -> None:
15          pass
16      @abstractmethod
17      def detach(self, observer: Observer) -> None:
18          pass
19      @abstractmethod
20      def notify(self) -> None:
21          pass
22  
23  # Конкретний суб'єкт
24  class ConcreteSubject(Subject):
25      def __init__(self):
26          self._observers: List[Observer] = []
27          self._state: str = ''
28  
29      def attach(self, observer: Observer) -> None:
30          # Реєстрація спостерігача
31          self._observers.append(observer)
32  
33      def detach(self, observer: Observer) -> None:
34          # Скасування реєстрації спостерігача
35          self._observers.remove(observer)
36  
37      def notify(self) -> None:
38          # Сповіщення усіх зареєстрованих спостерігачів
39          for observer in self._observers:
40              observer.update(self)
41  
42      @property
43      def state(self) -> str:
44          return self._state
45  
46      @state.setter
47      def state(self, value: str) -> None:
48          self._state = value
49          # Автоматичне сповіщення при зміні стану
50          self.notify()
51  
52  # Конкретний спостерігач A
53  class ConcreteObserverA(Observer):
54      def __init__(self, name: str):
55          self._name = name
56          self._observer_state: str = ''
57  
58      def update(self, subject: Subject) -> None:
59          # Отримання нового стану від суб'єкта
60          self._observer_state = subject.state
61          print(f'{self._name} отримав оновлення: {self._observer_state}')
62  
63  # Конкретний спостерігач B
64  class ConcreteObserverB(Observer):
65      def __init__(self, name: str):
66          self._name = name
67  
68      def update(self, subject: Subject) -> None:
69          print(f'{self._name} реагує на стан: "{subject.state}".upper()')
70  
71  # ── Демонстрація використання ──────────────────────────
72  if __name__ == '__main__':
73      subject = ConcreteSubject()
74  
75      observer_a = ConcreteObserverA('Спостерігач A')
76      observer_b = ConcreteObserverB('Спостерігач B')
77  
78      # Реєстрація спостерігачів
79      subject.attach(observer_a)
80      subject.attach(observer_b)
81  
82      # Перша зміна стану
83      print('--- Зміна стану 1 ---')
84      subject.state = 'активний'
85  
86      # Скасування реєстрації observer_b
87      subject.detach(observer_b)
88  
89      # Друга зміна стану (тільки observer_a отримає)
90      print('--- Зміна стану 2 ---')
91      subject.state = 'неактивний'
